# Question 27: Explain the rule of three/five/zero.

**Explain the rule of three/five/zero.**
    - **Answer/Explanation:** Rule of three: if a class needs destructor, copy constructor, or copy assignment, it needs all three. Rule of five adds move operations. Rule of zero delegates to smart pointers.
    - **Code Example:** Implement copy/move operations or use smart pointers.
    - **Key Concepts:** Resource management, copy semantics.
    - **Problem-Solving Technique:** Proper class design.
